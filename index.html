Gildas Le Louarn

Merchant adoption

How to foster Payment Request adoption?

Merchant value proposition

Pro

  • A way to access different payment means using unique API
  • Browser base payment experience may reduce friction and improve security

Cons

  • Implementation fragmentation, can't yet rely on to aggregate payment means

Payer value proposition

Pro

  • Consistent browser payment experience
  • Payment instrument storage simplify checkout experience

Cons

  • Low merchant support

Network and acquirer value proposition

Pro

  • Security of card acquisition rely on browser
  • Homogeneous integration maturity level

Cons

  • Security concerns rely on not yet deployed capabilities

PSP value proposition

Pro

  • Simpler way to provides payment methods
  • Opportunities to provide new aggregation APIs

Cons

  • Browser implementation not mature enough

Industry priorities

Whether or not related work is already happening within W3C, what are industry priorities?

Actors priorities

Identify one priority by actor:

  • Merchant, conversion rate
    require mature support
  • Payer, ease of use and trust
    require browser confidence
  • Network and acquirer, security
    require tokenization and authentication
  • PSP, easy to integrate solution
    require wide functional coverage

The end

May I conclude with personal thought?
